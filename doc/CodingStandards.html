Codeing standards

When coding in this project developers should try follow these standards.
Indents should be 4 spaces, no tabs.
Brackets for function blocks, if, for, while, and switch statements should be on a new line.
Brackets may be on the same line for empty code blocks.
Even single line code blocks must be in brackets. 
// Incorrect, brackets on same line.
void test() {
    for(int i = 0;i < 10;i++) {
        std::cout << i << endl;
    }
}
// Incorrect, no brackets.
void test() {
    for(int i = 0;i < 10;i++)
        std::cout << i << endl;
}
// Correct
void test()
{
    for(int i = 0;i < 10;i++)
    {
        std::cout << i << endl;
    }
}
Class and structure names should be in upper camel case.
Function names in should be in lower camel case.
Defines should be in all upper case.
Class member functions should be in lower camel case with a 'm_' appended to the beginning.
ALL destructor's will be declared virtual.
#define LIMIT 100
class TestClass
{
public:
    TestClass();
    virtual ~TestClass();
    
    void testFunction();
    void emptyTestFunction() {}; // Correct, empty code block.
    int BadFunctionName(); // Incorrect, first letter is upper case.
    
private:
    bool m_stateReady;
    int m_numberOfValues;
    double m_BadValueName; // Incorrect, first letter of name is upper case.
};
